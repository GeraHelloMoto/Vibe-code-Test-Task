# Repair Requests Service

Тестовое задание: веб-сервис для приема заявок в ремонтную службу.

## Запуск проекта

1. Убедитесь, что установлены Docker и Docker Compose.
2. Клонируйте репозиторий.
3. Выполните:   docker-compose up

4. Откройте в браузере:
- Создание заявки: http://localhost:8000/static/index.html
- Панель диспетчера: http://localhost:8000/static/dispatcher.html
- Панель мастера: http://localhost:8000/static/master.html

## Тестовые пользователи

В базе данных созданы:
- Диспетчер: ID 1 (логин не используется)
- Мастер Петров: ID 2
- Мастер Сидоров: ID 3

На странице мастера можно выбрать нужного мастера из выпадающего списка.

## Проверка гонки (конкурентного взятия заявки)

Выполните скрипт `race_test.sh` (предварительно убедитесь, что сервер запущен и заявка с ID 2 назначена на мастера 2):

```bash
chmod +x race_test.sh
./race_test.sh
```

## Автотесты
Запустите тесты внутри контейнера:

```bash
docker-compose exec web pytest tests/
```
